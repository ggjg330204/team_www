# 향후 계획 및 참고 (Future Plans & References)

본 문서는 현재 배포되지 않은 서비스, 향후 고도화 계획, 그리고 문서 업데이트 가이드를 포함합니다.

---

## 1. 미배포 서비스 및 제한사항

### 1.1 VPN Gateway (가상 사설망 게이트웨이)
*   **상태**: ❌ **미배포**
*   **사유**: 현재 Pure Cloud 환경으로 운영 중이며, 온프레미스 연동 요구사항이 없음. 비용 최적화(월 $150-500 절감).
*   **향후 계획**: 하이브리드 클라우드 전환 시 도입 검토.

### 1.2 DDoS Protection Standard
*   **상태**: ❌ **미배포** (Basic 사용 중)
*   **사유**: 월 $3,000에 달하는 높은 비용. 현재 규모에서는 Basic Tier와 WAF로 충분함.

### 1.3 Azure AD Premium 기능
구독 라이선스 제한으로 인해 다음 기능들은 사용할 수 없거나 대체 기능을 사용합니다.

| 기능 | 상태 | 대안 |
|:---|:---:|:---|
| **MFA (Multi-Factor Authentication)** | ❌ 불가 | 보안 기본값(Security Defaults) 사용 |
| **PIM (Privileged Identity Management)** | ❌ 불가 | 수동 역할 할당 관리 |
| **조건부 액세스 (Conditional Access)** | ❌ 불가 | NSG/방화벽으로 IP 제한 |

### 1.4 NSG Flow Logs
*   **상태**: ❌ **생성 불가**
*   **사유**: Azure에서 2025년 6월 30일부터 신규 NSG Flow Logs 생성 중단. 2027년 9월 30일 완전 폐기 예정.
*   **대안**: VNet Flow Logs 또는 Defender for Servers 네트워크 위협 탐지 사용.
*   **참고**: [Azure 공식 문서](https://learn.microsoft.com/azure/network-watcher/nsg-flow-logs-migrate)

### 1.5 멀티 리전 Geo-Routing
*   **상태**: ❌ **미배포**
*   **사유**: 학습용 Azure 구독(Student Subscription)의 SKU 제한으로 인해 멀티 리전(US West, Japan East) 리소스 생성이 불가능함.
*   **향후 계획**: 엔터프라이즈 구독 전환 시 즉시 도입 가능하도록 Terraform 모듈화 완료.

### 1.6 수동 활성화 생략 항목 (Sentinel Connectors)
Terraform으로 자동화할 수 없으며, 학생/조직 계정 권한(Global Admin) 제한이나 Azure 레거시 호환성 문제로 인해 수동 연결도 불가능한 커넥터입니다.

*   **Microsoft Entra ID (구 Azure AD)**
    *   **상태**: ❌ **생략 (비활성화)**
    *   **사유**: `Tenant Permissions` (Global Administrator/Security Administrator) 권한 부족. 해당 권한은 학생용 구독에서 일반적으로 부여되지 않음.
*   **Microsoft Defender for Cloud**
    *   **상태**: ❌ **생략 (비활성화)**
    *   **사유**:
        1.  **Legacy 버전**: 구독 호환성 문제로 `NotFound` 오류 발생.
        2.  **Tenant-based 버전**: Entra ID와 동일한 Tenant 레벨 관리자 권한 부족.
*   **Microsoft Entra ID Security Defaults (MFA 대체)**
    *   **상태**: ❌ **생략 (비활성화)**
    *   **사유**: `Enable security defaults` 설정 시 `Tenant Permissions` 부족으로 인한 접근 거부 발생.
*   **Insider Risk Management (Microsoft 365 내부 위협)**
    *   **상태**: ❌ **생략 (비활성화)**
    *   **사유**: 
        1.  **라이선스 필요**: Microsoft 365 E5, Microsoft 365 E5 Compliance, Microsoft 365 A5 등.
        2.  **테넌트 권한**: Global Administrator/Security Administrator 권한 부족.

## 2. 향후 고도화 계획

### 2.1 컨테이너 오케스트레이션 (AKS)
*   **목표**: VMSS 기반 애플리케이션을 Kubernetes(AKS)로 마이그레이션.
*   **기대 효과**: 배포 자동화(GitOps), 리소스 효율성 증대, 마이크로서비스 아키텍처 전환.

### 2.2 데이터 분석 플랫폼 (Synapse Analytics)
*   **목표**: MySQL 및 로그 데이터를 Synapse로 통합하여 비즈니스 인사이트 도출.
*   **구성**: Data Factory ETL → Synapse Dedicated Pool → Power BI 시각화.

### 2.4 CI/CD 파이프라인 구축
*   **목표**: GitHub Actions 또는 Azure DevOps를 도입하여 코드 커밋부터 배포까지 완전 자동화.
*   **현재**: Terraform 인프라 배포는 자동화되었으나, 앱 배포는 일부 수동.
